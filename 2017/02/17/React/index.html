将React项目push到git中总结

npm run dist编译生成代码到dist目录
git add dist/git commit -m "add dist"
git subtree push --prefix=dist origin gh-pages
prefix用来推送本地的dist目录，推送成功就可以通过github提供的web服务来访问站点：https://CuteCodevx.github.io/gallery-by-react
部署后问题：找不到app.js文件，在index.html中script标签的引入有问题，改了src地址,去掉‘\’,变成相对路径，在default.js中publicPath也需去掉‘\’

遇到的问题：使用yeoman无法生成gruntfile.js文件：这是因为generator-react-webpackV2.0移除了Grunt，可以执行npm start或者npm run serve启动服务,grunt:serve:dist/grunt build可用npm run dist来代替编译生成代码到dist目录,可看package.json文件

react特点

1.JSX语法
    html结构与js可以混在一起写，更加清晰
2.虚拟DOM
    通过DOM diff算法，只会更新有差异化的部分，不用渲染整个页面，从而提高效率
3.组件化
    把页面上的功能都写个组件，然后可以结合起来，可以做到重用
在页面中需要引入：react.js              核心代码
                react-dom.js            渲染DOM
                browser.min.js        解析JSX语法

ReactDOM.render()

React的最基本方法，将React的模板语法转为真正的HTML，并插入到指定的DOM节点里，再渲染它
    参数1：创建的元素
    参数2：要添加给谁（DOM元素）
    参数3：渲染成功后的回调函数

JSX语法

遇到尖括号开头的解析成html。遇到大括号开头的解析成JavaScript
可以在模板里插入变量，如果这个变量是个数组，则会自己把数组里所有数据全部显示
