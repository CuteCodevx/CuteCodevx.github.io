自适应rem布局

rem：相对于根节点字体大小的值 默认情况下16px=1rem；
使用rem布局时，应动态的修正字体的大小来兼容不同的分辨率，让所有的用rem单位的子元素跟着一起缩放，从而达到自适应的效果。
var doc=document.documentElement,
    resizeEvent='orientationchange' in window ? 'orientationchange' : 'resize';
recalc=function(){
    doc.style.fontSize=20*(doc.clientWidth/320)+'px';
};
window.addEventListener(resizeEvent,recalc,false);
document.addEventListener('DOMContentLoaded',recalc,false);

git github

git常用命令

把项目放到本地

1.在本地新建文件夹
2.新建文件夹下打开个git bash
3.运行命令 git clone 项目地址 如https：//github.com/username/repositoryname.git

把本地文件添加到github上

1.初始化本地仓库
git init           把这个目录变成git可以管理的仓库
2.添加到暂存区
    git add 文件名/.  （将本地文件加到仓库里）.代表全部文件
3.添加到版本区
    git commit -m "注释"         （设置提交信息）
    git commit -a -m '注释'        从本地直接到版本区，跳过了暂存区
4.本地仓库链接远程仓库
    git remote         查看远程仓库名称
    git remote -v     查看远程仓库地址
    git remote add origin git@github.com:username/repositoryname.git
5.push文件到仓库中
    git push origin master

把本地文件添加到github上小结

我在将本地文件push到github上时，首先在github上建立了一个新的repository，然后在本地的文件夹里执行git init；git add .;git commit -m "submit";并实时用git　status查看仓库的当前状态；然后我用了git push 命令,但却出现了failed to push some refs to 'https://github.com/dummymare/Hello-World.git'错误，查阅资料后知道了这是由于远程仓库有更新（因为建立它时我选择添加readme文件）但本地中没有，所以更新失败，此时我运行命令git pull origin master,但是依旧出现错误：fatal: refusing to merge unrelated histories，查阅资料后得知在git版本合并两个不同的项目，应当添加--allow-unrelated-histories这个命令，如：git pull origin master ----allow-unrelated-histories，这时就可看到本地文件中多出了一个readme文件，说明pull成功了，此时在运行git push origin master就可以了。这就是解决冲突后才可以push，这个步骤可能会提示要输入密码，输入注册时候的用户名与密码就可以。


如果从github上clone了一个项目到本地，然后想把这个项目重新push到另一个仓库中，那么需要删掉项目里的隐藏文件.git，然后重新init。

查看文件差异

查看本地与暂存区的差异：git diff
查看暂存区与版本区的差异：git diff--cached
查看本地与版本区的差异:git diff master

撤销

从暂存区撤消到本地：git reset HEAD 文件名
    在乱改了工作区(本地文件夹)，还添加到暂存区时使用，可以把暂存区的修改撤销掉重新放回工作区
把版本区的文件还原到本地：git checkout -- 文件名 
    可以丢弃工作区的修改，让这个文件回到最近一次git commit或git add的状态

删除

删除暂存区的文件：git rm 文件名
删除本地文件和暂存区文件：git rm -f 文件名
只删除暂存区，不删除本地文件：git rm --cached 文件名

恢复

恢复单个文件：git checkout 编码 文件名 
    编码通过git log（查看历史记录 git log --pretty=oneline能看着整齐些）去找
恢复整个版本的文件：git reset --hard 编码
查找历史纪录方便回滚：git reflog
    记录每次命令，可以找到commit id是什么，然后git reset --hard xxxid 就回来了

多人开发出现冲突

New collaborator添加开发人员
1.git fetch                           把线上的代码先下载下来
2.git diff master origin/master     查看冲突
3.git merge origin/master            将不同的代码显示到页面中，人为的判断用谁的代码
4.解决后把代码push上去

分支

git branch                  查看当前分支
git branch  新的分支名        新建分支
git checkout 分支名          切换分支
git checkout -b 分支名       快速新建并切换
git merge 分支名             合并分支
git branch -d 分支名         删除分支

bug分支

git stash 把当前工作现场储藏起来，再跳回当前分支 git checkout 分支名
git stash list 查看刚才的工作现场，
    恢复：1.git stash apply 恢复后 stash内容并不删除，需用git stash drop来删除；
         2.git stash pop 恢复的同时把stash的内容也删了

Flex(弹性)布局

容器的属性(父级)

flex-direction

项目的排列方向，主轴的方向
4个值：row         (默认)主轴为水平方向，从左到右 
      row-reverse      从右到左 
      column           主轴为垂直方向，从上到下
      column-reverse   从下到上

flex-wrap

3个值：nowrap      (默认)不换行，宽度若超过父级则自动缩小
      wrap              换行
      wrap-reverse      换行后原先的第一行在下方

flex-flow

flex-direction与flex-wrap的简写：默认值为 row nowrap；

justify-content

项目在主轴上的对齐方向
5个值：flex-start (默认)左对齐
      flex-end        右对齐
      center          居中 
      space-between   两端对齐，中间间隔相等
      space-around      每个项目两侧间隔相等

align-items

垂直对齐
5个值：flex-start    交叉轴起点对齐
      flex-end      交叉轴终点对齐
      center        中点对齐
      stretch       如果项目未设置高度，将占满整个容器的高度
      baseline      项目的第一行文字的基线(文字底部)对齐

align-content

多行对齐
6个值：flex-start
      flex-end
      center
      space-between;
      space-around
      stretch         (默认)

项目的属性(子元素)

order

定义项目的排列顺序，数值越小，排列越靠前，默认为0

.item{order:<integer>;}

flex-grow

定义项目的放大比例，默认为0

.item{flex-grow:<number>;}

flex-shrink

定义项目的缩小比例，默认为1，当空间不足时，都将等比例缩小。如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。

flex-basis

在分配多余空间之前，项目占据的主轴空间，默认值为auto，即项目的本来大小

flex

是flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto
该属性有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。
后两个属性值可选。

align-self

允许单个项目有与其他项目不一样的对齐方式，默认auto

创建型API

createElement,createTextNode,cloneNode,createDocumentFragment
